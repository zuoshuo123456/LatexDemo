# 🚀 获取可分享的演示链接

## 目标
获得一个可以直接分享的 URL,比如:
```
https://latex-demo-abc123.vercel.app
```

别人点开就能看到演示,无需安装任何东西。

---

## 第一步: 推送到 GitHub（首次需要）

### 1.1 创建 GitHub 仓库

1. 访问 https://github.com/new
2. 仓库名称填写: `latex-demo`（或其他你喜欢的名字）
3. **重要**: 不要勾选 "Add a README file"
4. 点击 "Create repository"

### 1.2 推送代码

在 `LatexDemo` 目录打开命令行,执行:

```bash
# 初始化 Git（如果还没有）
git init

# 添加所有文件
git add .

# 创建提交
git commit -m "Initial commit: Markdown + LaTeX Demo"

# 添加远程仓库（替换成你的 GitHub 用户名）
git remote add origin https://github.com/你的用户名/latex-demo.git

# 推送到 GitHub
git branch -M main
git push -u origin main
```

**或者直接双击运行**: `deploy-now.bat`（会自动执行上述步骤）

---

## 第二步: 部署到 Vercel（3分钟）

### 2.1 登录 Vercel

1. 访问 https://vercel.com
2. 点击右上角 "Sign Up"（如果没账号）或 "Log In"
3. 选择 "Continue with GitHub"（用 GitHub 账号登录最方便）

### 2.2 导入项目

1. 登录后,点击 "Add New..." → "Project"
2. 在列表中找到你的 `latex-demo` 仓库
3. 点击 "Import"

### 2.3 配置部署

**保持所有默认设置**,直接点击 "Deploy"

- Framework Preset: Other
- Root Directory: `./`
- Build Command: (留空)
- Output Directory: (留空)

### 2.4 等待部署

- 部署过程约 1-2 分钟
- 你会看到一个进度条和日志
- 完成后会显示 "Congratulations!" 🎉

---

## 第三步: 获取并分享链接

### 3.1 复制链接

部署完成后,Vercel 会显示你的项目 URL:

```
https://latex-demo-xxx.vercel.app
```

点击 "Visit" 按钮或复制这个 URL。

### 3.2 验证演示

访问这个 URL,你应该看到:
1. 紫色渐变的加载动画
2. "Markdown + LaTeX 混合渲染修复" 标题
3. 加载完成后进入演示项目

### 3.3 分享给别人

现在你可以把这个 URL 分享给任何人:
- 发送到微信、邮件、文档
- 别人点开就能直接看到演示
- 无需安装、无需配置、无需登录

---

## 🎯 完整流程时间估算

- GitHub 创建仓库: 2 分钟
- 推送代码: 2 分钟
- Vercel 部署: 3 分钟
- **总计: 约 7 分钟**


## ❓ 常见问题

### Q: 我没有 GitHub 账号怎么办?
A: 访问 https://github.com/signup 免费注册一个,只需要邮箱。

### Q: 我没有 Vercel 账号怎么办?
A: 用 GitHub 账号登录 Vercel 即可,不需要单独注册。

### Q: 部署是免费的吗?
A: 是的,Vercel 对个人项目完全免费,没有任何费用。

### Q: 链接会过期吗?
A: 不会,链接永久有效,除非你主动删除项目。

### Q: 可以自定义域名吗?
A: 可以,在 Vercel 项目设置中可以绑定自己的域名。
